# 细节玩法与 UI 交互逻辑

## 1. 玩家身份：监控者 (The Overseer)
玩家不在世界之内，而是通过一套复杂的监控系统（HUD）来干预和观察方舟。

## 2. 核心 UI 布局 (HUD Layout)
我们将弃用旧有的固定大小 UI，建立一套响应式全屏界面：

### A. 顶部状态中心 (Top Info Bar)
- **环境传感器**: 显示当前 Z 层坐标（e.g., `Z-Level: -10 (The Depths)`）。
- **时间指示器**: 游戏时间、季节、以及当前的模拟倍速（Play/Pause, 1x, 5x, 20x）。
- **层级切换提示**: 显式的上下翻层按钮。

### B. 右侧详情面板 (The Inspector Pane)
- **选中反馈**: 点击任何单位或物体，面板会从边缘平滑滑出。
- **单位面板**: 实时显示：
    - 性格 (Traits): 例如 "懒惰", "嗜血"。
    - 动机 (Motivations): 当前为什么想睡觉，为什么想打架。
    - 需求条 (Needs Bars): 饥饿、疲劳、舒适度的实时动态曲线。

### C. 左下角事件日志 (Telemetry Feed)
- 以控制台风格显示简报（Logs）。
- 重要事件（如：遭遇怪物、资源枯竭）会闪烁红色，点击日志可使摄像机自动跳转并切层至发生地。

## 3. 互动玩法：区域规划 (Zoning)
玩家不直接操作，但可以“标记”：
- **资源区**: 标记某层某处可采掘。
- **居住区**: 标记何处为禁行区或核心保护区。
- **优先级设置**: 在 UI 菜单中调整：`“食物采集 > 建筑维修”`，AI 将根据该权重自动接单。
